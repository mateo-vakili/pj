Evaluation

Initially we tried to test the html file with the atom browser and google chrome, We noticed that as the atom browser was smaller it was struggling with borders mainly because we used a double boarder style and also a thick width which would probably increase the size of sections more than 100%, (made the width of the border to 100 percent and had the mentioned problem both on chrome and atom and safari, we fixed the issue by making the value of width equal to auto so it adapts itself automatically). We tested the print file first with atom by just changing the href to print.css (stylesheet used for print) and in chrome we only had to press ctrl+p to see the details about the print preview. we had the same probelm in relation to the borders as half of it wouldnt be printed due to the thickness of borders exeeding the maximum width of the page while the width is equal to 100 and again it was solved by adding width:auto.

We checked the responsiveness of the webpage using atoms and chrome browsers and safari's developing tools. When we decreased the width it would successfully load the second stylesheet which was designed for small screen with a different design for our table.
